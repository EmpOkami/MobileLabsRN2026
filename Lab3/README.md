## Інструкція запуску

### Кроки

**1. Клонувати репозиторій**
```bash
git clone https://github.com/MobileLabsRN2026/lab3.git
cd lab3
```

**2. Встановити залежності**
```bash
npm install
```

**3. Запустити проєкт**
```bash
npx expo start
```

**4. Відкрити на пристрої**

- **Android / iOS** — відскануй QR-код у застосунку **Expo Go**
- **Емулятор Android** — натисни `a` у терміналі
- **Симулятор iOS** — натисни `i` у терміналі

> При першому запуску або після змін у `babel.config.js` використовуй:
> ```bash
> npx expo start --clear
> ```

## Реалізований функціонал

### Головний екран — Клікер

Інтерактивний об'єкт (кнопка «TAP ME») підтримує **6 типів жестів**:

| Жест | Бібліотечний тип | Нараховує |
|---|---|---|
| Одиночний тап | `Gesture.Tap()` | +1 очко |
| Подвійний тап | `Gesture.Tap().numberOfTaps(2)` | +2 очки |
| Довге натискання (1с) | `Gesture.LongPress()` | +5 очок |
| Перетягування | `Gesture.Pan()` | переміщення об'єкта |
| Свайп вправо/вліво | `Gesture.Fling()` | +1–10 очок (рандом) |
| Свайп вгору | `Gesture.Fling()` | +3 очки |
| Pinch (зведення пальців) | `Gesture.Pinch()` | +3 очки |

Усі жести об'єднані через `Gesture.Simultaneous()` та `Gesture.Exclusive()` і обробляються одним `GestureDetector`.

---

### Екран завдань

Містить **9 завдань** з відображенням прогресу та статусу виконання:

1. Зробити **10 кліків** — лічильник з прогрес-баром
2. Зробити **подвійний клік 5 разів**
3. **Утримати** об'єкт 3 секунди (LongPress)
4. **Перетягнути** об'єкт по екрану (Pan)
5. **Свайп вправо** (Fling Right)
6. **Свайп вліво** (Fling Left)
7. **Змінити розмір** об'єкта (Pinch)
8. Набрати **100 очок**
9. **Власне завдання** — свайп вгору 3 рази

Кожне завдання відображає:
- Іконку типу жесту
- Назву та опис
- Прогрес-бар (для числових завдань)
- Статус ✓ / ○

### Екран налаштувань

- Перемикач **темної / світлої теми** (зберігається в контексті)
- Статистика: поточна кількість очок та виконаних завдань

| Технологія | Призначення |

## Висновки

У ході виконання лабораторної роботи було розроблено мобільний застосунок-клікер з використанням жестових взаємодій на базі React Native та Expo.

**Що вивчено:**

- Роботу з бібліотекою `react-native-gesture-handler` — зокрема з новим Gesture API (`GestureDetector`, `Gesture.Tap()`, `Gesture.Fling()`, `Gesture.Pan()`, `Gesture.Pinch()`, `Gesture.LongPress()`), який є сучаснішим і стабільнішим порівняно зі старим підходом на основі компонентів-обгорток.

- Комбінування кількох жестів через `Gesture.Simultaneous()`, `Gesture.Exclusive()` та `Gesture.Race()` для коректної паралельної або пріоритетної обробки різних типів взаємодії.

- Використання `react-native-reanimated` для плавних анімацій через `useSharedValue` та `useAnimatedStyle`, а також виклик JavaScript-функцій зі worklet-контексту через `runOnJS()`.

- Побудову навігації між екранами за допомогою `@react-navigation/bottom-tabs` та управління глобальним станом через React Context API.

- Реалізацію підтримки світлої та темної теми з динамічним застосуванням стилів.

**Труднощі та їх вирішення:**

Основною складністю стала сумісність версій пакетів — зокрема `react-native-reanimated` вимагає встановлення через `npx expo install` для автоматичного підбору версії, сумісної з поточним Expo SDK. Також вкладені обробники жестів старої API конфліктували між собою, що було вирішено переходом на новий `GestureDetector`-підхід.
