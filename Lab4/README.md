# Лабораторна робота №4 — Файловий менеджер

## Тема

Робота з файловою системою в React Native з використанням бібліотеки expo-file-system.

## Мета

Розробити мобільний застосунок «Файловий менеджер», який реалізує навігацію по локальній файловій системі, операції створення, читання, редагування та видалення файлів і папок, перегляд детальної інформації про об'єкти файлової системи, а також відображення статистики використання пам'яті пристрою.

---

## Інструкція запуску

### Вимоги

- Node.js версії 18 або вище
- npm або yarn
- Expo CLI
- Додаток Expo Go на мобільному пристрої (Android або iOS)

### Кроки

1. Клонувати репозиторій:

```bash
git clone https://github.com/MobileLabsRN2026/lab4.git
cd lab4
```

2. Встановити залежності:

```bash
npm install
```

3. Встановити необхідні пакети Expo:

```bash
npx expo install expo-file-system
npx expo install react-native-screens react-native-safe-area-context
npm install @react-navigation/native @react-navigation/stack
```

4. Запустити проєкт:

```bash
npx expo start
```

5. Відсканувати QR-код у терміналі за допомогою додатку Expo Go на мобільному пристрої.

### Примітка щодо версії expo-file-system

У версії expo-file-system v54 і вище стандартний імпорт `expo-file-system` є застарілим. У цьому проєкті використовується легасі-імпорт:

```js
import * as FileSystem from 'expo-file-system/legacy';
```

## Опис реалізованого функціоналу

### 1. Навігація по файловій системі

Застосунок відкриває кореневу директорію `documentDirectory` бібліотеки `expo-file-system/legacy`. Вміст кожної директорії відображається у вигляді списку за допомогою компонента `FlatList`. Папки відображаються першими, файли — після них. При натисканні на папку відбувається перехід у вкладену директорію з оновленням заголовка навігації та індикатора поточного шляху (breadcrumb). Повернення до попередньої директорії реалізовано через стандартний механізм стекової навігації `@react-navigation/stack`.

### 2. Створення файлів і папок

Кнопка створення розміщена у правому нижньому куті екрана (FAB). Після натискання відкривається модальне вікно, в якому користувач обирає тип об'єкта — папка або текстовий файл (.txt), вводить назву та, у випадку файлу, початковий вміст. Папки створюються за допомогою `makeDirectoryAsync`, файли — за допомогою `writeAsStringAsync`.

### 3. Читання файлів

При натисканні на текстовий файл (.txt) відбувається перехід на екран редагування, де завантажується вміст файлу за допомогою `readAsStringAsync`. Вміст відображається у полі введення, доступному для редагування.

### 4. Редагування файлів

На екрані редагування користувач може змінювати вміст текстового файлу. Після внесення змін стає активною кнопка збереження. Збереження виконується через `writeAsStringAsync` з перезаписом файлу. Після успішного збереження відображається підтвердження.

### 5. Видалення файлів і папок

Видалення доступне через кнопку на кожному елементі списку або через довгий тап на елементі. Перед видаленням відображається діалог підтвердження через `Alert`. Видалення виконується за допомогою `deleteAsync` з параметром `idempotent: true`.

### 6. Перегляд детальної інформації

При натисканні на кнопку інформації навпроти файлу або папки відкривається модальне вікно з такими атрибутами:

- Назва файлу або папки
- Тип (за розширенням або «Папка»)
- Розмір у відформатованому вигляді (Б, КБ, МБ, ГБ)
- Дата останньої модифікації у форматі локалі `uk-UA`
- Повний шлях до об'єкта

Дані отримуються через `getInfoAsync` з параметром `size: true`.

### 7. Статистика використання пам'яті

На головному екрані відображається блок статистики пам'яті пристрою, який включає:

- Загальний обсяг пам'яті пристрою (`getTotalDiskCapacityAsync`)
- Обсяг вільного простору (`getFreeDiskStorageAsync`)
- Обсяг зайнятого простору (обчислюється як різниця)

Додатково відображається графічна смужка заповненості. Дані можна оновити вручну через кнопку оновлення.
